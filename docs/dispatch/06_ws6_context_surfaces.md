# WS-6 Context surfaces (dispatch brief)

Objective: wire the six existing mockups in as navigable context so the demo feels like a whole operating system, not one feature. Near free, because the mockups already exist and are polished.

Depends on: WS-0 only. Can run early, in parallel with everything.

Read first: `README.md`. Source mockups are in this repo at `mox_html_original/`: `mox_01_command (1).html`, `mox_02_extension (1).html`, `mox_03_manage (1).html`, `mox_04_invest (1).html`, `mox_05_build.html`, `mox_06_flywheel_diagrams.html`. Rename to clean slugs (mox_01_command, etc.) when importing.

Deliverables:
- Import the six mockups as navigable routes/views in the frontend.
- Light reskin for consistency where cheap (the deal name, the building, the Mox brand are already in them). Do not rebuild them.
- They are seeded-static; no live engine needed.
- Make them reachable from the adaptive command so the surface reads as one OS. The operating beats (the leak, repair history, monthly close) live here on `mox_03_manage`, since the real asset is a ground-up redevelopment without operating history.

Acceptance:
- All six are reachable and navigable.
- They read as visually consistent with the hero surfaces.
- They are clearly secondary: context for scope, not the focus of function.

Guardrails (hard):
- Any numbers in the mockups are illustrative placeholders; ensure nothing reads as earned-calibrated where it is not (guardrail 1). Label them representative.

Repos to reference: none. The mockups are in the doc-set collateral.
